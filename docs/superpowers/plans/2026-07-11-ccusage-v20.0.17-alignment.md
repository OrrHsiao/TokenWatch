# ccusage v20.0.17 Alignment Implementation Plan

> **2026-07-12 final-review amendment:** 原计划已按章执行；最终广域审查又确认四个独立缺陷。本 amendment 允许窄化修复 Codex 坏 timestamp 时的 total baseline（不改 replay 行选择），并要求 OpenCode 展示 fallback 与计价模型分离、两份定价 catalog 独立加载、上游 source provenance 与实际 bundle/生产映射 artifact SHA-256 分开固定。以下“Codex adapter 不改”和“只记录 source SHA”限制在这些窄化修复范围内被本 amendment 覆盖。

> **2026-07-12 fast artifact follow-up:** builtin fast multiplier 改为从同一 exact/prefix override 映射派生，移除六处重复 literal；production canonical 升级为带 schema 的 v2 序列化，artifact SHA-256 更新为 `148f61be850408d66bc146201a663d5a6edbd75db1a7e554e16958da26b0ee69`。

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让 TokenWatch 在当前支持的数据源范围内与官方 `ccusage v20.0.17 --offline` 对齐：保留已一致的 Codex replay 行选择，展开 Claude advisor usage，并让 Dashboard 与 Sessions 使用不重复加入 reasoning 的 token total，同时升级固定计价契约元数据。

**Architecture:** provider adapter 继续输出统一的 `ParsedUsageEntry`。Codex adapter 不改；Claude adapter 在主记录成功归一化后，使用独立窄解码器从同一顶层 JSON 行展开零到多条 advisor entries；统一 `TokenUsage.aggregateTotalTokens` 定义非重复总量，由 Dashboard 与 Sessions 两个累加器共同消费。离线价格资源和算法保持原样，只把 fixture 契约从 v20.0.16 升级到源码与资源均已审计的 v20.0.17。

**Tech Stack:** Swift 6、Foundation、Swift Testing、Xcode 26.5、macOS app-hosted unit tests、内嵌 JSON 定价资源、Homebrew `ccusage 20.0.17`、`jq`/`diff` 本地诊断；不增加运行时网络或第三方依赖。

```swift
private func makeEntry(
    provider: ProviderID,
    sessionID: String,
    timestamp: Date?,
    model: String = "test-model",
    input: Int,
    output: Int,
    cacheRead: Int = 0,
    cacheCreation: Int = 0,
    reasoning: Int = 0,
    cwd: String? = nil,
    isSubagent: Bool = false,
    upstreamProviderID: String? = nil
) -> ParsedUsageEntry {
    let suffix = [
        provider.rawValue,
        sessionID,
        "\(timestamp?.timeIntervalSince1970 ?? -1)",
        model,
        "\(input)",
        "\(output)",
        "\(cacheRead)",
        "\(cacheCreation)",
        "\(reasoning)",
        cwd ?? "nil",
        upstreamProviderID ?? "nil",
    ].joined(separator: "-")

    return ParsedUsageEntry(
        recordUUID: "record-\(suffix)",
        messageId: "message-\(suffix)",
        requestId: nil,
        sessionID: sessionID,
        timestamp: timestamp,
        model: model,
        cwd: cwd,
        agentId: nil,
        usage: TokenUsage(
            inputTokens: input,
            cacheCreationInputTokens: cacheCreation,
            cacheReadInputTokens: cacheRead,
            outputTokens: output,
            reasoningTokens: reasoning,
            serverToolUse: ServerToolUse(webSearchRequests: 0, webFetchRequests: 0),
            serviceTier: "",
            cacheCreation: CacheCreation(
                ephemeral1hInputTokens: 0,
                ephemeral5mInputTokens: cacheCreation
            ),
            inferenceGeo: "",
            iterations: [],
            speed: ""
        ),
        isSubagent: isSubagent,
        provider: provider,
        upstreamProviderID: upstreamProviderID,
        upstreamCost: nil
    )
}
```

- [ ] **Step 5: 在 ClaudeMessage.swift 增加独立窄 advisor 解码器**

在 `ClaudeBillingCacheCreation` 后增加下列类型。它们不得加入 `ClaudeBillingMessage` 或 `ClaudeBillingUsage.CodingKeys`，从类型边界上保证主 usage 解码不依赖 iterations：

```swift
/// 顶层 Claude iteration 中可单独计费的 advisor usage。
struct ClaudeAdvisorUsage: Sendable {
    let model: String
    let usage: ClaudeBillingUsage
}

/// 独立解码顶层 message.usage.iterations；失败时由调用方只忽略 advisors。
enum ClaudeAdvisorUsageDecoder {
    private static let marker = Data(#""advisor_message""#.utf8)

    static func decode(
        from lineData: Data,
        using decoder: JSONDecoder
    ) -> [ClaudeAdvisorUsage] {
        guard lineData.range(of: marker) != nil,
              let envelope = try? decoder.decode(
                  ClaudeAdvisorEnvelope.self,
                  from: lineData
              ) else {
            return []
        }
        return envelope.message.usage.iterations.compactMap(\.advisorUsage)
    }
}

private struct ClaudeAdvisorEnvelope: Decodable {
    let message: Message

    struct Message: Decodable {
        let usage: Usage
    }

    struct Usage: Decodable {
        let iterations: [Iteration]
    }

    struct Iteration: Decodable {
        let advisorUsage: ClaudeAdvisorUsage?

        private enum CodingKeys: String, CodingKey {
            case type, model
        }

        init(from decoder: Decoder) throws {
            let container = try decoder.container(keyedBy: CodingKeys.self)
            let type = try container.decode(String.self, forKey: .type)
            let model = try container.decodeIfPresent(String.self, forKey: .model)
            let usage = try ClaudeBillingUsage(from: decoder)
            guard type == "advisor_message",
                  let model,
                  !model.isEmpty else {
                advisorUsage = nil
                return
            }
            advisorUsage = ClaudeAdvisorUsage(
                model: model,
                usage: usage
            )
        }
    }
}
```

每个 iteration 都先按 flattened `ClaudeBillingUsage` 解码，再按 type/model 过滤。因此任意 iteration（包括普通类型或空 model）的 token 类型非法都会使整个 advisor envelope 返回空数组；主 decoder 已经在另一条路径成功，因此主 entry 仍保留。

- [ ] **Step 3: 创建自包含 Codex token/cost diagnostic**

`.build/` 在 clean checkout 中可能为空；不要依赖旧 ignored 文件。用 `apply_patch` 新建 `.build/live_usage_diagnostic.swift`（若已存在则完整替换），最终完整内容必须是：

```swift
import Foundation

private struct DiagnosticDay: Codable {
    let date: String
    var inputTokens: Int
    var cacheCreationTokens: Int
    var cacheReadTokens: Int
    var outputTokens: Int
    var reasoningOutputTokens: Int
    var totalTokens: Int
    var costUSD: Double
}

private struct DiagnosticReport: Codable {
    let daily: [DiagnosticDay]
}

@main
private enum LiveUsageDiagnostic {
    static func main() throws {
        guard CommandLine.arguments.count == 2,
              let timeZone = TimeZone(identifier: "Asia/Shanghai") else {
            throw DiagnosticError.invalidArguments
        }

        let codexRoot = URL(fileURLWithPath: CommandLine.arguments[1])
        let files = try CodexRolloutScanner().scanAll(in: codexRoot)
        let parser = CodexRolloutParser()
        let pricingSpeed = CodexServiceTierResolver().pricingSpeed(at: codexRoot)
        let entries: [ParsedUsageEntry]
        if ProcessInfo.processInfo.environment["TW_PER_FILE"] == "1" {
            entries = try files.flatMap {
                try parser.parseFile($0, pricingSpeed: pricingSpeed)
            }
        } else {
            entries = try parser.parseAllFiles(files, pricingSpeed: pricingSpeed)
        }

        let repositoryRoot = URL(
            fileURLWithPath: FileManager.default.currentDirectoryPath
        )
        let pricingRoot = repositoryRoot.appendingPathComponent("TokenWatch/Pricing")
        let liteLLM = try LiteLLMPriceCatalog(
            data: Data(
                contentsOf: pricingRoot.appendingPathComponent("litellm_prices.json")
            )
        )
        let modelsDev = try ModelsDevPriceCatalog(
            data: Data(
                contentsOf: pricingRoot.appendingPathComponent("models-dev-pricing.json")
            )
        )
        let costResolver = UsageCostResolver(
            pricingEngine: PricingEngine(
                pricingTable: PricingTable(
                    liteLLMEntries: liteLLM.entries,
                    modelsDevEntries: modelsDev.entries
                )
            )
        )

        var calendar = Calendar(identifier: .gregorian)
        calendar.timeZone = timeZone
        var totals: [String: DiagnosticDay] = [:]

        for entry in entries {
            guard let timestamp = entry.timestamp else { continue }
            let components = calendar.dateComponents(
                [.year, .month, .day],
                from: timestamp
            )
            guard let year = components.year,
                  let month = components.month,
                  let day = components.day else {
                continue
            }
            let key = String(format: "%04d-%02d-%02d", year, month, day)
            guard key >= "2026-07-08", key <= "2026-07-11" else {
                continue
            }

            var value = totals[key] ?? DiagnosticDay(
                date: key,
                inputTokens: 0,
                cacheCreationTokens: 0,
                cacheReadTokens: 0,
                outputTokens: 0,
                reasoningOutputTokens: 0,
                totalTokens: 0,
                costUSD: 0
            )
            value.inputTokens = value.inputTokens.addingSaturated(
                entry.usage.inputTokens
            )
            value.cacheCreationTokens = value.cacheCreationTokens.addingSaturated(
                entry.usage.totalCacheCreationTokens
            )
            value.cacheReadTokens = value.cacheReadTokens.addingSaturated(
                entry.usage.cacheReadInputTokens
            )
            value.outputTokens = value.outputTokens.addingSaturated(
                entry.usage.outputTokens
            )
            value.reasoningOutputTokens = value.reasoningOutputTokens.addingSaturated(
                entry.usage.reasoningTokens
            )
            value.totalTokens = value.totalTokens.addingSaturated(
                entry.usage.aggregateTotalTokens
            )
            value.costUSD += costResolver.resolvedCost(for: entry)
            totals[key] = value
        }

        let report = DiagnosticReport(
            daily: totals.values.sorted { $0.date < $1.date }
        )
        let encoder = JSONEncoder()
        encoder.outputFormatting = [.prettyPrinted, .sortedKeys]
        FileHandle.standardOutput.write(try encoder.encode(report))
        FileHandle.standardOutput.write(Data([0x0A]))
    }
}

private enum DiagnosticError: Error {
    case invalidArguments
}
```

该文件只存在于 ignored `.build/`；不要把它加入 Xcode target、production bundle 或 Git index。
