# 会话表格默认窗口双向免滚动设计

## 背景

会话页在默认 `1180 × 840pt` 主窗口中展示 10 条记录时，用户仍需向下滚动才能看到第 10 行和分页栏；表格也需要横向滚动才能看到全部列。

上一版修复把横向滚动条的 `17pt` 避让空间加入了表格 document 高度。新截图与真实运行调查表明，这会让 legacy 样式下的 clip view 比 document 矮 `17pt`，内层表格因此产生纵向滚动范围。横向手势带有纵向分量时，表头或底部内容会被推出视口。与此同时，当前行内容宽度为 `1,108pt`，而默认窗口实际只提供 `880pt` 的表格宽度，所以横向滚动是必然的。

## 目标

默认窗口 `1180 × 840pt` 下同时满足：

1. 完整表头、10 条数据行、分页栏均在初始纵向视口内，不需要上下滚动。
2. 时间、会话 ID、工具、项目、主模型、总 Token、成本、记录数八列均在初始横向视口内，不需要左右滚动。
3. overlay 与 legacy 两种横向滚动条样式都不能产生内层纵向滚动范围，也不能覆盖分页栏。
4. 小于默认尺寸的窗口继续允许外层纵向滚动和表格横向滚动。

## 非目标

- 不改变每页 10 条、分页模型、排序方式或数据聚合。
- 不改变默认主窗口尺寸、侧边栏宽度及其他页面布局。
- 不缩小现有 48pt 数据行、44pt 表头和 44pt 分页栏高度。
- 不移除小窗口所需的滚动能力。

## 方案

### 1. 默认宽度内压缩表格

默认表格视口宽度为：

`1180 - 244（侧边栏）- 2 × 28（页面水平边距）= 880pt`

表格行采用以下紧凑宽度：

| 列 | 宽度 |
| --- | ---: |
| 最近时间 | 120pt |
| 会话 ID | 150pt |
| 工具 | 84pt |
| 项目 | 132pt |
| 主模型 | 116pt |
| 总 Token | 86pt |
| 成本 | 76pt |
| 记录数 | 68pt |

八列合计 `832pt`；列间距从 `8pt` 调整为 `4pt`，7 个间距共 `28pt`；行左右内边距从 `16pt` 调整为 `10pt`，合计 `20pt`。最终总宽度正好为 `880pt`。

会话 ID 保留现有 `150pt`，避免复制按钮标题和图标再次被压缩不可见。其他较长字段继续使用现有单行截断；会话 ID 仍可复制完整值。

表格最小 document 宽度同步改为 `880pt`。默认窗口下 document 与 clip view 等宽，没有横向滚动范围；窗口变窄后 document 保持最小宽度，横向滚动自然恢复。

### 2. 滚动条 gutter 与 document 分离

表格真实内容高度保持：

`44（表头）+ 10 × 48（数据行）+ 44（分页栏）= 568pt`

滚动条 gutter 继续通过 overlay/legacy 两种 regular scroller 宽度的最大值动态计算，本机结果为 `17pt`。滚动视图外壳高度仍为 `568 + 17 = 585pt`，但 document 高度只保留 `568pt`。专用的顶部对齐表格 document 使用稳定的坐标方向，保证：

- legacy 样式：clip view 高度为 `568pt`，与 document 完全相等；滚动条占外壳底部 `17pt`。
- overlay 样式：clip view 高度为 `585pt`，document 顶部对齐，底部多出的 `17pt` 承接滚动条。

因此两种样式下 document 都不会高于 clip view，内层纵向滚动范围恒为零。分页栏仍属于 document，继续与表格列保持一致的横向坐标。

### 3. 默认纵向预算

会话页继续使用上一版已确认的局部尺寸：

`20（顶部）+ 64（标题）+ 14 + 104（指标卡）+ 14 + 585（表格外壳）+ 20（底部）= 821pt`

默认可视高度为 `840pt`，保留 `19pt` 余量，因此外层页面不应产生纵向滚动范围。

## 测试设计

新增或改写回归测试，使用 21 条会话以覆盖 10 行、三页按钮和最宽分页控制组合，并在真实默认窗口尺寸下验证：

1. document 宽度不大于横向 clip view，可视区域包含完整表头和表格宽度。
2. overlay 与 legacy 样式下，document 高度都不大于 clip view；完整表头、第 10 行和分页栏均在内层可视区域。
3. 外层 document 不高于默认窗口可视区域，完整会话栈和分页栏均可见。
4. 小窗口仍产生预期的纵向和横向滚动范围。
5. 现有分页、会话 ID 可见性、行高及独立横向滚动相关测试继续通过。

测试必须先在当前实现上以“默认宽度仍横向溢出”及“legacy 内层仍有 17pt 纵向溢出”的原因失败，再实施生产代码修改。

## 验收标准

- 默认 `1180 × 840pt` 初始画面无需任何滚动即可同时看到八列表头、10 条数据行和分页栏。
- overlay/legacy 切换及横向手势不会裁切表头、第 10 行或分页栏。
- 小窗口仍可通过滚动访问全部内容。
- 完整单元测试与 Debug 构建通过，并以真实 Debug App 做一次未滚动截图复验。
